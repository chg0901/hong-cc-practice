# Context Management Rules — 上下文管理与 Prompt Cache 策略

## 核心原则

**缓存还热、任务没换，继续聊；缓存过期、任务切换、噪音太多，果断重开。**

## 6 个核心指令速查

| 指令 | 用途 | 缓存影响 | 使用时机 |
|------|------|----------|----------|
| `/context` | 查看 token 占用分布 | 只读，不影响缓存 | 每个会话开头、装新 skill/MCP 后 |
| `/resume` | 恢复上次会话 | 命中缓存（1hr 内），省 90% | 任务没变、距离上次 <1hr |
| `/rewind` | 回滚到任意消息 | 保留回滚点之前的缓存 | 方案失败、上下文被污染 |
| `/clear` | 彻底重开 | **完全重建**，~50K token 全价 | 任务切换、缓存过期、噪音太多 |
| `/compact` | 有损压缩 | **当前缓存失效**，用总结替换 | 阶段性完成时主动触发 |
| subagent | 隔离脏活 | 独立窗口（5min 缓存），主会话不受影响 | 只要结论不要过程的工作 |

## Prompt Cache 三条规则

### 1. 前缀匹配

缓存只认从头开始、一字不差的前缀。第 1 页改一个字，整份缓存作废。

Claude Code 输入布局（不变 → 变化）：

```
系统提示 + 工具定义 + CLAUDE.md  →  对话历史  →  新消息
[不变，天然命中缓存]              [增长中]      [全价]
```

### 2. 缓存窗口

| 上下文 | 缓存寿命 | 每次命中刷新 |
|--------|---------|------------|
| 主会话 | 1 小时 | 是 |
| 子代理 | 5 分钟 | 是 |

闲置超过窗口时间 → 缓存过期 → 下次全价重建。

### 3. 成本差异

- 缓存命中：**1/10 价格**（90% 节省）
- 缓存未命中：全价
- 基础设施（系统提示 + 工具 + CLAUDE.md）：约 **50,000 token**，每次 /clear 都要全价重建

## "继续聊 vs 开新会话"决策表

### 继续当前会话（任一满足）

- 任务没变（还在调同一个 bug / 写同一个模块 / 围绕同一组文件）
- 距离上一条消息不到 1 小时（缓存还活着）
- 上下文里的内容还在有用（之前读过的文件、讨论过的方案还在用）

### 果断开新会话（任一满足）

- 任务换了（从认证做到支付，上下文完全不同）
- 闲置超过 1 小时（缓存大概率过期）
- 上下文被噪音塞满（试了十几种方案、读了一堆无关文件，输出质量下滑）

## 三条反直觉规则

### 1. 频繁 /clear 不是省钱，是烧钱

每次新会话重建 ~50K token 基础设施全价。活跃会话中这 50K 一直躺在缓存里，只付 1/10。

**规则**：做完一个小任务不 /clear，只有满足上方"开新会话"条件时才 /clear。

### 2. 一次做对比来回改三轮便宜

复杂任务开着 thinking 一次搞定，比关掉 thinking 来回改三轮更省。每多一轮对话，整个上下文重新传输一次。

**规则**：简单任务用 `/effort low` 省钱；复杂重构保持 thinking 开启。

**Why**: 思考是一次性开销，上下文传输是每轮都要付的开销。

### 3. 给路径不贴内容

不要把一万行日志/代码粘贴进对话。给文件路径，让 Claude 自己用 Read/grep 提取相关行。

**规则**：长内容（日志、数据文件、大段代码）只给路径，不粘贴。

**Why**: 最便宜的 token 是那些根本没进上下文的 token。

## /rewind 进阶用法

1. 方案 A 失败 → **不要** 直接说"换个方案"
2. 先让 Claude `summarize from here` 做自我总结
3. /rewind 回到"文件读完、但还没试方案 A"的那一刻
4. 带着总结作为新 prompt 的开头重新下指令

```
# 错误：直接纠正
"刚才不对，换 B 试试"          ← 方案 A 的脏上下文仍在

# 正确：summarize → rewind → 新 prompt
1. "summarize from here: 方案 A 失败，原因是 X"
2. /rewind 回到读文件后
3. "别用 A，foo module 没暴露那个接口，直接上 B"
```

## /compact 进阶用法

- 默认：让 Claude 自己总结，lossy 压缩
- 带参数：`/compact focus on auth refactor, drop test debugging`
- **主动 compact** > 被动自动压缩（自动压缩时 Claude 可能砍掉重要信息）
- 时机：阶段性完成、缓存还热时主动执行

## 模型切换注意

- 缓存按模型隔离：Opus 积累的缓存，切到 Sonnet 后全部失效
- 需要轻量模型时，用 subagent 而非切主模型
- 便宜模型（如 GLM/Kimi）混用时，/exit 切换配置后 --resume 可以复用上下文

## CLAUDE.md 精简建议

- 官方建议控制在 **200 行以内**
- 只保留真正长期有效的规则
- 代码审查流程、迁移步骤等"只在特定时刻需要"的内容 → 挪到 skills（按需加载）
- 本项目现状：29 个 rules 文件分担了 CLAUDE.md 的负担，使其保持在合理范围
