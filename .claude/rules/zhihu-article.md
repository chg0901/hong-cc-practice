# Zhihu Article Markdown Formatting Rules

## Scope

适用于 `docs/zhihu_*.md` 系列文件（面向知乎平台发布的文章）。

## 核心规则：两级标题

知乎编辑器仅渲染 `#`（一级）和 `##`（二级）两种标题。`###` 及以下不会渲染为标题。

因此，所有知乎文章**只使用两级标题**：

| 层级 | Markdown | 用途 | 示例 |
|------|----------|------|------|
| 一级 | `# 标题` | 文章总标题 + 所有主要章节标题 | `# 背景`、`# 一、为什么需要`、`# 教训与最佳实践` |
| 二级 | `## 小标题` | 章节内的段落小标题 | `## 第一步：确认错误来源`、`## 1. Prompt Hooks 不要加 JSON 约束` |

## 标题层级映射

对于普通 markdown 文档中的标题层级，在知乎文章中做如下映射：

| 普通 markdown | 知乎文章 | 说明 |
|--------------|---------|------|
| `# 标题` | `# 标题` | 文章标题，保持不变 |
| `## 背景` / `## 一、xxx` | `# 背景` / `# 一、xxx` | **主章节提升为一级标题** |
| `### 子标题` | `## 子标题` | **段落内小标题降为二级** |
| `#### 更细的标题` | `## 更细的标题` 或加粗文本 | 全部用二级或改用加粗 |

## 标题命名规范

1. **主章节标题**使用中文数字编号：`# 一、xxx`、`# 二、xxx`、`# 三、xxx`
2. **无编号章节**使用描述性标题：`# 背景`、`# 根因分析`、`# 一句话总结`
3. **步骤类子标题**使用序号：`## 第一步：xxx`、`## 第二步：xxx`
4. **列表类子标题**使用数字：`## 1. xxx`、`## 2. xxx`

## 其他知乎格式注意事项

1. **不要用 `---` 水平线**：知乎不支持 `---` 渲染为分隔线，用空行替代
2. **图片用 `<img>` 标签控制宽度**：`<img src="path" width="600"/>` 比纯 markdown `![]()` 更可控
3. **代码块必须标注语言**：` ```json `、` ```bash ` 等，知乎会做语法高亮
4. **表格正常使用**：知乎支持标准 markdown 表格
5. **引用块 `>` 正常使用**：知乎支持引用块渲染
6. **对比图用 HTML table 并列**：两张相关图片（如正常流程 vs 故障流程）用 `<table>` 并排展示，方便对比
7. **摘要图（Abstract Image）**：每篇知乎文章开头（标题之后、`# 背景` 之前）应有一张总结全文核心内容的图，帮助读者快速理解文章主题

## 摘要图规范

每篇 `docs/zhihu_*.md` 文章必须在标题后紧跟一张摘要图：

```markdown
# 文章标题

> 一句话摘要

<img src="../mermaid/figure/xxx_abstract.png" width="600"/>

- 图：[xxx_abstract.png](../mermaid/figure/xxx_abstract.png)
- 源：[xxx_abstract.mmd](../mermaid/text/xxx_abstract.mmd)

# 背景
...
```

摘要图应概括全文的核心流程/架构/对比关系，让读者在 5 秒内理解文章主题。

## 判断文件是否为知乎文章

文件名匹配 `docs/zhihu_*.md` 模式的文件，自动适用本规则。
