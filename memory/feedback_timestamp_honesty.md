**规则**：所有工作日志（work_summary、CLAUDE.md Change Log、changelog.md）中的时间戳 `[HH:MM:SS]` 必须使用 `date '+%H:%M:%S'` 获取的实际主机时间，**绝对不能编造或估算**。

**Why**：2026-04-16 会话中，实际主机时间是 09:38，但日志中编造了 10:00~12:00 的未来时间戳。用户明确指出这是严重错误，违反了之前已写入的规则。编造时间戳使日志不可追溯，失去审计价值。

**How to apply**：
1. 每写一个 ChangeLog 条目前，先执行 `date '+%H:%M:%S'` 获取当前时间
2. 同一天跨多个任务时，每个任务完成时获取一次时间
3. 不得"回推"或"估算"时间——用你执行 `date` 命令的那一刻的真实时间
4. 如果忘记获取时间，使用日志最后一条记录之后的时间（如 `date` 当前值），而不是编造
5. 回顾性补写日志时，标注"补记"并使用当前时间
